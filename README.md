명령형 프로그래밍 방식으로 코딩했던것을 선언형으로 바꿔봤다.
코드가 거대해지고 UI의 업데이트가 많아질 경우, 어느 지점, 어느 시점에 DOM을 업데이트 했느냐를 추적하기가 힘들어진다. 
따라서 DOM에 접근하고 업데이트하는 시점에 대한 명확한 기준점을 만들자.
어떠한 상태를 기준으로 랜더링 하도록 만들자.
